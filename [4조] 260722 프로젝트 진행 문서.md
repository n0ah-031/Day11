# [4조] 260722 프로젝트 진행 문서

## 1. 프로젝트 제목
> **취합앱** — 양식 생성부터 검토·취합까지 자동화하는 자료 취합 전용 웹 서비스

## 2. 프로젝트 소개
- **무엇을 만드나:** AI 양식 생성 → 회신 파일 검토 → 엑셀·한글(hwpx) 취합·병합을 한 번에 처리하는 웹 서비스 (현재는 프론트엔드 단독 데모 SPA)
- **누구를 위해:** 여러 부서·개인의 산출물을 모아 취합하는 기관 내 실무 담당자 (최대 약 500명 규모)
- **왜 (어떤 문제를 해결):** 양식이 제각각인 회신 파일을 수작업으로 검토·병합하며 발생하는 누락·중복·형식 불일치 문제를 자동 검토와 취합 자동화로 해결

## 3. 사용자 흐름
> 로그인(사번) → 홈(생성/취합 탭) → [생성] AI 문답형 양식 생성 → 양식 다운로드
> → [취합] 회신 파일 업로드 → 검토 결과(오류/경고/정보 배지) 확인 → 오류 리포트 다운로드 → 합성 모드(A~D) 선택 → 취합 실행 → 결과 다운로드 → 이력 조회
> ※ Admin: 계정 관리·정책 설정·시스템 대시보드

`[여기에 이미지 첨부]`

## 4. 기술 스택
- **프론트엔드:** React 18 + Vite + TypeScript, React Router, Zustand(persist→localStorage), Tailwind CSS
- **백엔드:** 데모는 백엔드 없음(프론트 단독). 정식 버전 설계: Python FastAPI + Supabase(Postgres·Storage·Auth)
- **데이터베이스 / 기타:** 데모는 localStorage. 엑셀 처리는 SheetJS(xlsx)로 실제 클라이언트 로직 구현
- **사용 도구·서비스:** Claude Code(계획 기반 에이전트 개발), Vitest(단위 테스트), OpenAI GPT API(정식 버전 예정, 데모는 문답 목업)

## 5. 진행 상황
- **전체 단계:** 기획(PRD) – 설계(design) – 데모 개발 – 통합 검증 – 데모 시연 – (이후 백엔드 확장)
- **현재 위치:** 데모 개발 완료 직전 — 계획된 16개 작업 중 15개 완료, 최종 통합 검증(Task 16) 진행 중
- **완료된 것:**
  - PRD v1.3, 설계 문서(design.md) 확정
  - PRD 10개 화면 전체 구현: 로그인, 홈(생성/취합 탭), AI 문답형 양식 생성, 엑셀 업로드, 검토 결과, 합성 모드 선택, 결과, hwpx 병합, 이력, Admin 콘솔
  - 엑셀 파이프라인은 실제 동작: 파싱 → 규칙 기반 검토(오류/경고/정보 배지) → 합성 모드 A~D 취합 → 결과·오류 리포트 xlsx 생성 (+ 모듈별 단위 테스트 통과)

- **진행 중인 것:** Task 16 최종 통합 검증 — 전체 라우트 점검, 빌드·테스트 일괄 실행, 골든 패스 수동 워크스루(로그인→양식 생성→업로드→검토→취합→이력→Admin→hwpx), 반응형 확인
