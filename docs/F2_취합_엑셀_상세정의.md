## 1. 개요

### 1.1 목적 및 범위
본 문서는 PRD F2-1~F2-12(§4 "F2. 취합 — 엑셀")의 구현 상세를 정의한다. PRD가 규정한 파이프라인(업로드 → ①검토 → ②이상판별 → ③범위확인 → ④전처리 → ⑤합성모드선택 → ⑥합성 → 다운로드) 중, 특히 **①검토 단계의 데이터 유형별 판정 로직과 AI 연동 아키텍처**를 중심으로 다룬다. F3(한글 취합), F1(AI 양식 생성)은 본 문서 범위 밖이며 필요한 경우 참조만 한다.

### 1.2 데이터 유형 정의
엑셀 셀 데이터는 다음 3종으로 분류한다.

| 유형 | 정의 | 예시 |
|---|---|---|
| 정성적(텍스트) | 문자열 값. 코드표 기반/패턴 기반/자유서술형으로 세분 | 부서명, 성명, 비고 |
| 정량적(숫자) | 수치 값 | 금액, 수량, 예산 집행률 |
| 이미지 | 셀에 임베디드된 이미지 객체(floating object) | 증빙사진, 서명 이미지 |

### 1.3 검증 원칙 (핵심 아키텍처)
- **정성적·정량적 데이터**는 다음 2단계 파이프라인을 거친다.
  1. **1단계 — 규칙기반 알고리즘 검증**: attribute(컬럼 속성) 기준 값 유효성을 규칙으로 판정
  2. **2단계 — AI(LLM) 재검증**: **파일 전체가 1단계를 통과한 경우에만** 그 파일의 정성적·정량적 데이터를 대상으로 GPT API를 통해 속성-값 정합성을 재추론(파일 단위 게이팅, §4.2). 파일 내 단 하나의 셀이라도 1단계 위반이 있으면 그 파일은 2단계로 진입하지 않는다
  3. 두 단계 모두 통과해야 최종 **"정상"**. 어느 단계에서든 위반 발견 시 **"이상"** 확정 (상세 로직은 §4)
- **이미지 데이터**는 1단계(규칙기반)만 수행하고 종료한다. AI Vision 기반 내용 분석은 본 범위에서 제외하며 v2 백로그로 남긴다(§14).
- 이 원칙은 PRD §6.4("F2는 AI 장애와 무관하게 규칙기반으로 정상 동작")에 대한 부분 예외를 발생시키므로, §13에서 예외 규정과 장애 폴백을 별도로 정의한다.

### 1.4 처리 시나리오 (단계별 흐름)

1. 사용자가 취합할 실적자료 엑셀 파일을 업로드한다 (§2)
2. 1단계 알고리즘으로 파일 내 모든 셀을 끝까지 스캔하여 누락/정합성/중복/충돌을 검토한다(첫 위반 셀에서 검토를 중단하지 않음) (§4.1, §5)
   - 파일 내 위반 셀 없음 → 2단계로 진행
   - 파일 내 위반 셀 1개 이상 → **파일 전체**가 즉시 이상판별 화면(§6)에 '이상' 상태로 표시(위반 셀 전부를 파일명·시트명·셀 위치까지 안내), 해당 파일은 2단계로 넘어가지 않음(불필요한 API 호출 방지)
3. **파일 전체가 1단계를 통과한 경우에만** 그 파일의 정성적·정량적 데이터를 AI(LLM)로 재검증한다(파일 단위 게이팅) (§4.2)
   - 이상 없음 → 이상판별 화면에 '정상' 표시, 다음 단계(체크박스 선택) 진행 가능
   - 이상 있음(개별 셀 부적합 판정 또는 AI 재검증 실패) → 이상판별 화면에 '이상' 표시, 파일명·시트명·셀 위치까지 안내
4. 이상판별 화면에 정상/이상 결과를 출력한다 (§6)
   - 파일명별로 정상/이상 상태만 표시하고, 이상인 경우 상태값에 마우스오버/클릭 시 사유를 노출한다
   - 알고리즘 단계에서 발견됐는지 AI 단계에서 발견됐는지는 구분하지 않고 통합 표시한다(내부 등급은 §6.2 참조)
5. 사용자가 체크박스로 취합할 자료를 선택한다 (§7)
6. 합성 방식(모드 A/B/C/D)을 선택한다 (§9)
7. 취합을 실행하고, 결과 자료를 미리보기한 뒤 다운로드한다 (§10)

## 2. 업로드 및 구조 인식 (F2-1)

| 항목 | 규칙 |
|---|---|
| 지원 포맷 | `.xlsx`만 지원 (구버전 `.xls` 미지원) |
| 시트 인식 | 워크북 내 전체 시트 열거. 숨김 시트는 기본 제외하며, 필요 시 사용자가 포함 옵션 선택 가능 |
| 헤더 인식 | 상단부터 스캔하여 연속된 텍스트 행 중 데이터 행과 구분되는 첫 행을 헤더로 추정(제목행 오인식 방지 규칙은 §2.2 참조). 병합 셀이 포함된 경우 병합 영역의 최좌상단 값을 헤더명으로 사용 |
| 데이터 영역 인식 | 헤더 다음 행부터 연속된 데이터 블록을 데이터 영역으로 인식. 중간의 완전 공백 행은 데이터 종료로 간주하지 않고 건너뜀(기본값 2행까지 허용, 초과 시 종료로 판단 — 조정 가능, §2.2 참조) |
| 이미지 anchor 인식 | xlsx의 이미지는 셀 값이 아니라 `xl/drawings/drawing*.xml`의 floating object로 저장됨. anchor의 from(시작)~to(끝) 셀 좌표를 모두 파싱해 전체 점유 범위를 소속 범위로 확정(단일 셀로 축약하지 않음). 범위-병합 셀 정합성 검증은 §5.3 참조 |
| 구조 인식 실패 | 세부 사유별 처리는 §2.1 참조. 해당 파일/시트는 검토를 진행하지 않고 이상판별 화면에 "이상"으로 즉시 표시 |

### 2.1 업로드/구조 인식 오류 안내 메시지

업로드 및 구조 인식 단계에서 발생 가능한 오류는 "구조 인식 실패"로 뭉뚱그리지 않고 사유별로 세분화하여, 웹 앱 팝업으로 즉시 안내한다. 각 메시지에는 사용자가 바로 조치할 수 있도록 해결방안을 함께 포함한다.

| 상황 | 판정 시점 | 팝업 메시지 예시 |
|---|---|---|
| 포맷 자체가 틀림(.xls, .xlsm, .csv 등) | 업로드 즉시(구조 인식 이전) | "지원하지 않는 파일 형식입니다. .xlsx 파일만 업로드할 수 있습니다.\n해결방법: 엑셀에서 '다른 이름으로 저장' → 파일 형식을 'Excel 통합 문서(*.xlsx)'로 변경한 뒤 다시 업로드해주세요." |
| 포맷은 .xlsx이나 헤더 행을 찾을 수 없음 | 구조 인식 단계 | "{파일명}의 {시트명} 시트에서 헤더(열 제목) 행을 찾을 수 없습니다.\n해결방법: 제목·로고가 있는 행과 실제 데이터 사이에 명확한 열 제목 행이 있는지 확인해주세요." |
| 시트가 완전히 비어있음 | 구조 인식 단계 | "{파일명}의 {시트명} 시트에 데이터가 없습니다.\n해결방법: 빈 시트를 삭제하거나 데이터를 입력한 뒤 다시 업로드해주세요." |
| 헤더는 있으나 데이터 행이 하나도 없음 | 구조 인식 단계 | "{파일명}의 {시트명} 시트에 헤더는 있으나 입력된 데이터가 없습니다.\n해결방법: 데이터를 입력한 뒤 다시 업로드해주세요." |
| 파일 손상/암호 보호로 열람 불가 | 업로드 직후 파싱 시도 | "{파일명}을 열 수 없습니다(손상되었거나 암호로 보호된 파일일 수 있습니다).\n해결방법: 엑셀에서 파일이 정상적으로 열리는지, 암호가 걸려있지 않은지 확인 후 다시 업로드해주세요." |

- {파일명}/{시트명}은 실제 업로드된 값으로 치환한다.
- 파일/시트 단위 오류처럼 셀 위치가 의미 없는 경우, 이상판별 화면(§6.3)의 "파일명·시트명·셀 위치·사유" 표시 포맷에서 셀 위치란은 "(해당없음)"으로 표기한다.
  - 예: `A파일 - Sheet1 - (해당없음): 헤더 행을 찾을 수 없음`
  - 예: `B파일 - (해당없음) - (해당없음): 지원하지 않는 파일 형식(.xls)`

### 2.2 헤더/데이터 영역 인식 보완 규칙

**헤더 후보 판정 규칙 (제목행 오인식 방지)**
- 타이틀 행 skip: 값이 있는 셀이 전체 열 폭 대부분을 차지하는 단일 병합 셀 하나뿐인 행은 "제목행"으로 간주해 헤더 후보에서 제외한다.
- 다음 행 타입 대비 검증: 헤더 후보 행은 바로 다음 행이 데이터 타입(숫자/날짜 등 혼재)을 보여야 헤더로 확정한다. 다음 행이 공백이거나 후보 행과 동일하게 순수 텍스트면 후보에서 제외하고 다음 행으로 스캔을 계속한다.
- 예시: 1행(병합 제목) → 2행(공백) → 3행(부서명/담당자/예산액 등 헤더) → 4행(숫자 포함 데이터)의 경우, 1행은 두 규칙 모두에서 제외되고 3행이 헤더로 확정된다.

**공백 행 허용치 (조정 가능한 기본값)**
- 데이터 영역 중간의 연속 공백 행 허용치는 시스템 기본값 **2행**이며, 그룹 구분 공백을 3행 이상 쓰는 서식을 감안해 파일/양식 단위로 조정 가능하도록 설계한다(구현 시 템플릿 설정 값으로 관리 권장).
- 기본값 2행의 근거: 실무 서식에서 그룹 구분용 공백행이 통상 1~2행을 넘지 않는다는 경험적 가정.

## 3. 데이터 유형 분류 기준

컬럼(속성) 단위로 아래 우선순위에 따라 유형을 분류한다.

1. **F1 작성기준 스키마 우선 참조**: 양식 생성 단계(F1-5)에서 정의된 필드 유형이 있으면 그대로 사용
2. **스키마 부재 시 휴리스틱 분류** (값 타입 샘플링이 최종 근거, 헤더명 키워드는 1차 후보 추정용 보조 힌트):
   - 이미지 anchor가 존재하는 컬럼 → 이미지
   - 헤더명 키워드 매칭으로 1차 후보를 추정: 국문("날짜/금액/수량/비율" → 정량적 후보, "부서/성명/코드" → 정성적 코드표 기반 후보) + 영문("date/amount/count/rate/qty" → 정량적 후보, "dept/name/department/code/id" → 정성적 코드표 기반 후보)
   - 상위 N행(기본값 20행, 전체 데이터 행이 20행 미만이면 전체 행 — 시스템 기본값, 조정 가능) 값의 타입 분포를 샘플링하여 최종 판정한다. 키워드 후보와 값 타입 분포 결과가 다르면 **값 타입 분포를 최종 기준으로 채택**하며, 키워드 매칭에서 국/영문 모두 히트가 없으면 값 타입 분포만으로 판정한다
   - 정성적 컬럼 중 사전 정의 코드표가 없고 값의 길이/형태가 비정형이면 자유서술형으로 분류
3. **분류 불확실 시 기본값**: 자유서술형(가장 관대한 검증 기준)으로 처리하여 과탐지를 방지

## 4. 검토 파이프라인 아키텍처 (F2-2 공통 구조)

### 4.1 1단계 — 규칙기반 알고리즘 검증
- 대상: 모든 셀 (정성적/정량적/이미지)
- **파일 내 모든 셀을 끝까지 스캔한다**(첫 위반 셀 발견 시 전체 검토를 중단하지 않음). 위반이 발견된 모든 셀은 누락 없이 이상판별 화면 상세 목록(§6.3)에 노출되어야 한다
- 각 컬럼의 attribute(§5의 유형별 규칙)를 기준으로 값을 판정하되, 검증 순서는 **누락 → 정합성 → 중복/충돌**(동일 키 그룹 판정) 순으로 수행한다
  - 누락이면 정합성·중복/충돌 검사를 생략하고 "누락"으로 확정한다(빈 값에 다른 사유가 중복 표시되는 것을 방지)
  - 정합성은 셀 자체만으로 판정 가능한 검증이라 중복/충돌(동일 키 그룹핑이 필요한 상대적으로 무거운 연산)보다 먼저 수행한다
  - 중복과 충돌은 별도 단계가 아니라 "동일 키 그룹핑 → 그룹 내 값 비교"라는 하나의 판정 로직에서 나오는 두 분기다(§5.1)
- 위반이 발견된 셀은 즉시 "이상" 확정된다. 다만 **2단계(AI) 진입 여부는 개별 셀이 아니라 파일 단위로 결정**되므로(§4.2), 위반 셀이 있으면 그 셀만이 아니라 **파일 전체**가 2단계로 넘어가지 않는다
- 이미지는 1단계 결과가 곧 최종 결과다

### 4.2 2단계 — AI(LLM) 재검증 (정성적·정량적 데이터만)
- **대상 판정은 파일 단위**: 파일 내 모든 정성적·정량적·이미지 셀이 1단계를 전부 통과해야, 그 파일의 정성적·정량적 셀들이 2단계로 진입한다. 파일 내 단 하나의 셀이라도 1단계 위반이 있으면 파일 전체가 2단계로 진입하지 않고 이상판별 화면에 즉시 "이상"으로 표시된다(불필요한 API 호출 방지)
- 입력 컨텍스트: 해당 컬럼의 attribute 정의(F1 작성기준의 필드 설명/질문), 셀 값, 그리고 아래 기준에 따른 추가 컨텍스트
  - **인접 컬럼(같은 행) 포함 기준**: 자유서술형 텍스트, **코드표 기반 정성적 데이터**(마스터 목록엔 있으나 문맥상 다른 값이 맞는지 판단이 필요, §5.1)에는 같은 행의 관련 인접 컬럼 값을 함께 포함한다(예: "사업 추진 경과" 검증 시 "사업명"·"추진 기간" 값을 함께 전달해야 서술 내용이 실제 사업과 부합하는지 판단 가능). **패턴 기반** 정성적 데이터(정규식/문자 구성 검증만으로 충분, 예: 성명·이메일)는 해당 값만으로 판정 가능하므로 인접 컬럼을 포함하지 않는다
  - **컬럼 population 통계 포함 기준(정량적 데이터 전용)**: 정량적 데이터의 문맥상 이상치 판단은 인접 컬럼이 아니라 **같은 컬럼의 다른 행 값 분포**(평균·표준편차 또는 대표 샘플 값)가 필요하므로, 이를 별도 컨텍스트로 포함한다
- 자유서술형 텍스트의 경우, LLM 호출 전에 규칙기반 특징추출(키워드 중복도, 질문-답변 유형 매칭, 길이비율 등)을 수행하고 그 결과를 프롬프트 컨텍스트에 포함하여 LLM이 최종 판정을 내리도록 한다
- 정량적 데이터의 경우, 1단계로 못 잡는 문맥상 이상치(예: 형식·타입은 정상이나 다른 항목 대비 비합리적으로 큰/작은 값)를 보완적으로 점검
- LLM 응답은 정형화된 판정(적합/부적합) + 사유 텍스트로 파싱하여 저장
- 위반 판정 시 "이상" 확정(사유는 AI 응답 그대로 보관), 적합 판정 시에만 최종 "정상"
- 성능: 파일 전체가 1단계를 통과한 경우만 대상이므로 호출량이 제한적. 다건 셀을 배치로 묶어 1회 API 호출로 처리하는 것을 권장
- **배치 부분 실패 처리**: 배치(예: 셀 100개) 중 일부만 파싱 가능한 응답을 받은 경우, 배치 전체를 재시도하지 않는다. 정상 파싱된 항목은 그대로 판정을 반영하고, 파싱 실패한 항목만 최대 1회 개별 재시도한다. 재시도 후에도 실패하면 §4.2.1에 따라 처리한다

#### 4.2.1 AI 재검증 실패 처리 (개별 항목 파싱 실패)
- 서비스 전체 장애·타임아웃(§13.3)과는 별개로, 서비스는 정상이나 특정 항목의 LLM 응답이 파싱 불가능한 경우 해당 셀을 **"이상(AI 재검증 실패)"**으로 확정한다(§13.3의 장애 폴백과 달리 "정상(AI 미검증)"으로 처리하지 않음)
- 이상판별 화면에서 마우스오버/클릭 시 실패 사유를 상세히 노출한다. 예:
  > `A파일 - 실적_Sheet1 - D12셀: AI 재검증 실패`
  > 사유: "AI 응답을 해석할 수 없어 최종 판정을 내리지 못했습니다. (1단계 규칙기반 검증은 통과했으나, 2단계 AI 재검증에서 예상된 '적합/부적합' 형식이 아닌 응답을 받아 처리가 중단되었습니다.) 담당자가 직접 값을 확인해주세요."
- 서비스 전체 장애로 인해 2단계 자체가 생략된 경우는 이 규정이 아니라 §13.3(장애 폴백)을 따르며, 기존과 동일하게 "정상(AI 미검증)"으로 처리한다(PRD §6.4 원칙 유지)

### 4.3 이미지 예외
이미지는 1단계(규칙기반)만 수행하고 종료한다(§5.3). AI Vision 기반 내용 분석은 v2 백로그(§14)로 관리한다.

## 5. 데이터 유형별 검토 규칙 상세 (F2-2)

### 5.1 정성적(텍스트) 데이터

**누락 (전 하위유형 공통, 1단계 최우선 판정)**

| 항목 | 판정 기준 |
|---|---|
| 누락 | 셀 값이 빈 문자열이거나 공백문자만 존재 |

누락이 아닌 경우, 텍스트 컬럼은 3가지 하위 유형으로 나뉘며, 유형별로 attribute 정합성 판정 방식이 다르다.

| 텍스트 컬럼 유형 | 1단계 판정 방식 | 정상 | 경고(자동교정 가능) | 오류 |
|---|---|---|---|---|
| 코드표 기반 (부서명, 직급 등) | 사전 정의 마스터 목록 매칭 | 목록에 존재 | 공백·대소문자 차이만 존재(정규화 후 일치) | 목록에 없음 |
| 패턴 기반 (성명, 이메일 등) | 정규식/문자 구성 검증 | 패턴 일치 | 붙여쓰기·구분자 차이 등 자동 변환 가능 | 숫자·특수문자 등 이질적 문자 포함 |
| 자유서술형 (비고 등) | 길이·금칙어 규칙 체크만 | 제한 이내 | 길이 초과(자동 절삭 가능) | 금칙어 위반 |

**2단계(AI 재검증) 처리**
- 코드표 기반: 인접 컬럼 컨텍스트(§4.2)를 활용해 1단계로 걸러지지 않는 미묘한 오분류(예: 마스터 목록엔 있으나 문맥상 다른 부서가 맞는 경우)를 보완 점검
- 패턴 기반: 인접 컬럼 없이 해당 값 자체만으로, "패턴은 맞지만 실제로 유효한지"(예: 형식은 맞으나 부자연스러운 값)를 보완 점검
- 자유서술형: 규칙기반 특징추출(키워드 중복도, 질문-답변 유형 매칭, 길이비율) 결과를 LLM에 전달하여 "해당 컬럼이 묻는 내용에 실제로 부합하는 답변인지"를 최종 판정

**중복/충돌 (전 하위유형 공통, 1단계, 동일 키 그룹 판정)**

| 항목 | 판정 기준 |
|---|---|
| 중복 | 동일 키(예: 사번) 기준 행이 2회 이상 등장. 전 컬럼 값이 완전 동일하면 경고(자동 제거 가능), 일부만 동일하면 충돌로 분류 |
| 충돌 | 동일 키에 대해 값이 상이한 행이 존재 |

### 5.2 정량적(숫자) 데이터

| 검토 항목 | 1단계 규칙 | 정상 | 경고(자동교정 가능) | 오류 |
|---|---|---|---|---|
| 누락 | 셀 값 공란 | 값 존재 | — | 필수 필드 공란 |
| 정합성 | 타입(숫자 여부), 범위(min/max), 자릿수(소수점 규칙) | 규칙 전부 만족 | 콤마·통화기호 등 제거 후 규칙 만족 | 제거 후에도 숫자가 아니거나 범위 초과 |
| 중복 | 동일 키 행 중복 | 중복 없음 | 완전 중복 행(자동 제거 가능) | 키만 같고 값 상이(충돌로 분류) |
| 충돌 | 동일 키, 값 상이 | — | — | 발견 즉시 오류 |

**2단계(AI 재검증)**: 형식·범위는 정상이나 같은 항목의 다른 행 대비 문맥상 비합리적인 값(이상치)을 보완적으로 점검. 이상치 판단 시 "경고"로 분류(확정적 오류로 보지 않음).

### 5.3 이미지 데이터 (1단계에서 종료)

| 검토 항목 | 1단계 규칙(시스템 기본값) | 정상 | 경고 | 오류 |
|---|---|---|---|---|
| 누락 | 셀에 이미지 anchor(임베디드 객체) 존재 여부 | 객체 존재 | — | 필수 이미지 필드에 객체 없음 |
| 정합성 | 확장자 화이트리스트(jpg/jpeg/png), 용량 상한 10MB, 최소 해상도 200×200px | 전부 만족 | — | 화이트리스트 외 형식, 용량·해상도 미달 |
| 소속 범위 정합성 | anchor 전체 범위(from~to)가 병합 셀 영역과 일치하는지, 또는 단일 데이터 행(레코드) 경계 안에 있는지 확인 | 병합 영역과 일치하거나 단일 레코드 행 안에 있음 | 병합 셀이 아니면서 2개 이상의 데이터 행(레코드)에 걸침 — 소속 모호, 자동 처리 불가(사용자 확인 권고) | — |
| 중복 | 이미지 파일 해시(SHA-256) 완전 일치 | 해시 상이 | 동일 해시 파일 재사용 발견(자동 처리 불가, 사용자 확인 권고) | — |
| 충돌 | 동일 키(동일 행/개체)에 대해 필드당 허용 이미지 개수 정책 위반(예: 1장만 허용 필드에 2장 이상 존재) | 정책 이내 | — | 정책 위반 |

이미지는 내용(사진이 실제 증빙에 맞는지 등)을 판단하지 않으며, 구조/메타데이터 기준으로만 판정한다. AI Vision 기반 내용 분석은 v2 백로그(§14)로 유지한다.

### 5.4 작성기준(F1 스키마) 확장안

PRD F1-5의 작성기준 스키마(날짜/성명/금액/부서명 예시)를 아래와 같이 확장하여, 본 문서의 시스템 기본값을 필드별로 재정의할 수 있도록 한다.

| 확장 필드 | 적용 대상 | 예시 값 |
|---|---|---|
| `value_range` (min/max) | 정량적 | 금액: 0 이상, 상한 없음 |
| `decimal_places` | 정량적 | 0 (정수만 허용) |
| `master_list_ref` | 정성적(코드표 기반) | 부서 코드표 ID |
| `pattern_regex` | 정성적(패턴 기반) | 성명: `^[가-힣]{2,5}$` |
| `max_length` / `banned_words` | 정성적(자유서술형) | 최대 500자 |
| `allowed_extensions` | 이미지 | jpg, png |
| `max_file_size_mb` | 이미지 | 10 |
| `min_resolution` | 이미지 | 200×200 |
| `max_count_per_key` | 이미지 | 1 |

스키마에 값이 없으면 본 문서 §5의 시스템 기본값을 사용한다.

### 5.5 데이터유형 × 검토항목 × 검증단계 통합 매트릭스

| 데이터 유형 | 누락 | 정합성 | 소속범위 | 중복 | 충돌 | 2단계(AI) 적용 |
|---|---|---|---|---|---|---|
| 정성적 - 코드표 기반 | 1단계 | 1단계 | — | 1단계 | 1단계 | ✅ |
| 정성적 - 패턴 기반 | 1단계 | 1단계 | — | 1단계 | 1단계 | ✅ |
| 정성적 - 자유서술형 | 1단계 | 1단계(길이·금칙어) | — | 1단계 | 1단계 | ✅ (내용 정합성 판정) |
| 정량적 | 1단계 | 1단계 | — | 1단계 | 1단계 | ✅ (문맥상 이상치 보완) |
| 이미지 | 1단계 | 1단계 | 1단계(§5.3) | 1단계 | 1단계 | ❌ (v2 백로그) |

> "2단계(AI) 적용 ✅" 표시는 데이터 유형상 2단계 대상이 될 수 있다는 뜻이며, 실제 적용 여부는 **소속 파일이 1단계를 전부 통과했는지**(파일 단위 게이팅, §4.2)에 따라 결정된다.

## 6. 이상판별 화면 (F2-3)

### 6.1 화면 표시 원칙 — 정상/이상 2단계 표시
- 파일 리스트에는 파일명과 **정상 / 이상** 상태만 표시한다. PRD 원안의 "정상(녹색)/경고 N(황색)/오류 N(적색)" 3단 카운트 배지 대신, 화면 표기는 이진화한다
- "이상"이 1단계(알고리즘)에서 발견됐는지 2단계(AI)에서 발견됐는지는 화면에 구분 표시하지 않는다 — 사용자에게는 검증 단계가 아니라 "무엇이 문제인지"가 중요하기 때문
- 파일 내 검토 대상(여러 시트·여러 셀) 중 하나라도 "이상"이면 해당 파일 전체를 "이상"으로 표시한다

### 6.2 내부 판정 등급은 유지 (오류/경고)
- 화면 표시는 이진화하지만, 시스템 내부적으로는 §5의 오류/경고 구분을 그대로 유지한다
- 1단계에서 발견된 이상: PRD 기존 배지 등급 분류 기준에 따라 오류(취합 훼손 문제) 또는 경고(자동 처리 가능한 경미한 문제)로 분류
- 2단계(AI)에서 발견된 이상: 확정적이지 않은 LLM 판단이므로 기본적으로 **"경고"**로 분류한다(오탐 시 취합이 불필요하게 차단되는 것을 방지)
- **AI 재검증 실패**(§4.2.1, 서비스는 정상이나 개별 항목 응답 파싱 실패): 판정 자체가 불가능해 데이터 유효 여부를 알 수 없는 상태이므로 **"오류"**로 분류한다(§6.4의 "정상(AI 미검증)"과 달리, 검증 정보가 전혀 없는 상태를 경고로 취급해 조용히 취합에 포함되는 것을 방지)
- 이 내부 등급은 화면에 노출되지 않으며, F2-4의 체크박스 기본 선택 로직(§7)에만 사용된다

### 6.3 이상 상세 노출 (마우스오버 / 클릭)
- 파일 리스트에서 "이상" 상태값에 마우스를 올리거나 클릭하면, 해당 파일 내 모든 문제 항목을 목록으로 노출한다
- 각 항목은 **파일명 · 시트명 · 셀 위치 · 사유**를 포함한다 (예: "A파일 - A-1 시트 - C4셀: 필수값 누락")
- 사유 텍스트에는 참고용으로 내부 심각도를 함께 표기한다. 확정적 위반은 **[경미]**(예: "날짜 형식 자동 변환 가능") / **[심각]**(예: "필수값 누락")으로, 판정 자체가 불가능했던 경우는 **[확인필요]**(예: "AI 재검증 실패 — 사유: AI 응답 파싱 불가")로 구분 표기한다. 단, 검증 단계(1단계/2단계) 자체는 표기하지 않는다(§6.1)
- 한 파일에 문제가 여러 건이면 전체를 목록으로 나열한다
- **AI 재검증 실패 예시** (§4.2.1, [확인필요] 태그):
  > `A파일 - 실적_Sheet1 - D12셀: [확인필요] AI 재검증 실패`
  > 사유: "AI 응답을 해석할 수 없어 최종 판정을 내리지 못했습니다. (1단계 규칙기반 검증은 통과했으나, 2단계 AI 재검증에서 예상된 '적합/부적합' 형식이 아닌 응답을 받아 처리가 중단되었습니다.) 담당자가 직접 값을 확인해주세요."

### 6.4 AI 미검증 표시 (서비스 전체 장애 — 정상 처리)
- AI 재검증 단계가 API **서비스 전체** 장애·타임아웃으로 생략된 경우(§13.3), 해당 항목은 "정상" 상태에 보조 텍스트로 **"정상(AI 미검증)"**을 표시하여 1단계만으로 통과했음을 알린다
- 이는 서비스는 정상이나 개별 항목만 판정 실패한 경우(§4.2.1, "이상(AI 재검증 실패)")와는 다른 상태다 — 전자는 정책적 예외로 "정상" 취급, 후자는 §6.3의 [확인필요] 태그로 "이상" 취급하여 혼동하지 않도록 한다

## 7. 취합 범위 확인 (F2-4)

PRD F2-4 규칙을 그대로 따른다. 체크박스 기본 선택 상태는 §6.2의 내부 판정 등급(오류/경고/정상)을 기준으로 하며, 오류는 해제, 경고·정상은 체크 상태로 초기화한다. 화면에는 §6.1에 따라 정상/이상만 보이므로, 사용자 입장에서는 "이상"으로 표시된 파일 중 일부가 기본 체크되어 있는 것으로 보일 수 있다(내부적으로 경고 등급이기 때문). "AI 미검증" 보조표시는 기본 선택 여부에 영향을 주지 않는다.

**파일 내 등급 혼재 시 대표 등급 산정 규칙**: 한 파일 안에 여러 셀의 등급이 섞여 있으면(예: 오류 셀 1개 + 경고 셀 3개 + AI 재검증 실패 셀 1개), **가장 심각한 등급을 파일의 대표 등급으로 채택**한다(우선순위: 오류 > 경고 > 정상). 대표 등급이 체크박스 기본 선택 여부를 결정한다. 단, 마우스오버/클릭 시에는 대표 등급 하나만 보여주지 않고 §6.3에 따라 파일 내 혼재된 모든 등급과 각각의 사유를 전부 목록으로 노출한다.

## 8. 전처리 (F2-5)

- 전처리는 §7에서 사용자가 체크(선택)한 자료에 한해서만 적용된다. 체크 해제된 자료는 전처리 없이 취합 대상에서 제외된다
- 1단계에서 "경고"로 분류된 항목은 자동 교정 규칙(§5의 각 항목별 경고 조건에 대응하는 변환 로직: 형식 통일, 공백/구분자 정규화, 완전 중복 행 제거 등)을 적용한다
  - **중복 판정 범위**: 동일 키 중복/충돌 판정(§5.1, §5.2)은 **해당 파일 내부**로 한정한다. 서로 다른 파일 간 동일 레코드 중복은 본 범위에서 다루지 않으며, v2 백로그(§14)로 관리한다
- **2단계(AI)에서 발견된 이상은 자동 교정 대상에서 제외**한다. LLM 판단을 근거로 원본 데이터를 자동 변경하지 않으며, 항상 사용자 확인(F2-3 화면)을 거치도록 한다
- **감사 추적**: 자동 교정을 적용할 때는 원본 값을 함께 보존한다(§11 ReviewResult의 원본값/교정값 필드). 담당자가 추후 "어떤 값이 무엇으로 바뀌었는지" 취합 결과 또는 리포트에서 조회할 수 있도록 한다
- 이미지는 원본 포맷·해상도를 유지하며 자동 변환하지 않는다(경고만 표시)

## 9. 합성 모드 선택 및 합성 (F2-6~F2-10)

PRD 모드 A/B/C/D(§F2, 합성 모드 상세 정의 표)의 입출력 스펙을 그대로 따른다. 본 문서에서 추가로 정의할 사항은 다음과 같다.

- **이미지 포함 셀 병합**: 이미지는 셀 값이 아닌 floating object이므로, 합성 시 원본 anchor 좌표를 새 시트의 셀 좌표 기준으로 재계산하여 재배치해야 한다. 시트 구조가 바뀌는 모드(B/C/D)에서는 재계산 로직이 필수이며, 원본 그대로인 모드 A는 anchor 이동이 없다
  - **재배치 규칙**: 원본 anchor의 행/열 폭(크기)은 그대로 유지하고, 위치만 해당 레코드가 새 시트에서 배치되는 행 오프셋만큼 이동한다. 예: 원본에서 C6:D8(3행×2열) 크기였던 이미지가 세로 누적으로 다음 부서 블록 뒤에 이어 붙으면, 크기는 동일하게 유지한 채 C9:D11로 이동한다
- **오류 등급 항목 강제 포함 시 처리**: §7에서 사용자가 오류 등급 항목을 수동으로 재체크해 포함한 경우, 별도 검증·차단 없이 원본 값 그대로 합성에 포함한다. 모드 D의 SUMIF/SUMIFS 수식 범위에 비숫자 값이 섞여 수식이 깨지는 경우(`#VALUE!` 등)도 별도 처리하지 않으며, 사용자가 취합 결과를 검토하고 해당 셀을 직접 수정하도록 한다
- 모드 C 그룹 매핑, 모드 D 요약 지표 정의는 PRD 잔여 리스크 노트와 동일하게 미결 상태이며 §14에 재기재한다

## 10. 결과 리포트 및 다운로드 (F2-11, F2-12)

### 10.1 결과 미리보기 특이사항 안내 (F2-11)

§7·§9에서 사용자가 오류 등급 항목을 강제로 체크(포함)하여 취합한 경우, 다운로드 전 결과 미리보기 화면에 **"특이사항"** 안내 영역을 노출한다.

- **표시 조건**: 강제로 체크되어 포함된 오류 등급 ReviewResult가 1건 이상 존재하는 경우에만 노출
- **안내 문구 예시**: "강제로 포함하신 항목 중 일부가 형식 오류를 포함하고 있어, 결과 파일의 일부 값·수식이 정상적으로 계산되지 않았을 수 있습니다(예: `#VALUE!`). 아래 목록을 확인 후 원본 값을 수정해주세요."
- **목록 항목**: 파일명 · 시트명 · 셀 위치 · 사유(§10.2 오류 유형 참조)
- 이 안내는 다운로드를 막지 않는다(강제 포함은 사용자의 의도된 선택). 참고 정보로만 제공하며, 셀 자체에는 그대로 `#VALUE!` 등 실제 계산 결과가 표시된다(§9)

### 10.2 오류 리포트 다운로드 (F2-12)

오류 리포트(F2-12)는 xlsx 파일 하나에 시트 2개로 구성한다.

**시트 1: 오류 목록** (미해결 이상 항목)

| 컬럼 | 설명 |
|---|---|
| 파일명 / 시트 / 셀 위치 | 기존 PRD 정의 |
| 오류 유형 | 정합성/누락/중복/충돌/**소속범위**(이미지 전용, §5.3)/**AI 재검증 실패**(§4.2.1) |
| 검증 단계 | 1단계(규칙) / 2단계(AI) / **2단계(실패)**(AI 재검증 실패인 경우) |

| 교정값 | 자동 교정 후 값 |
| 적용 규칙 | 교정 사유(§5의 경고 조건) |

## 13. AI 연동 기술 사항

### 13.1 GPT API 연동
- 인증키는 환경변수(`OPENAI_API_KEY`)로 관리하며 `.env` 파일에 저장한다. 코드에 하드코딩 금지, `.env`는 버전관리 제외(.gitignore) 필수
- PRD §6.5의 어댑터 계층 원칙을 따라, 2단계 AI 재검증도 동일 어댑터를 통해 호출하여 추후 벤더 교체가 가능하도록 구현
- **배치 인터페이스**: F1의 단건 질의응답형 인터페이스와 별개로, 어댑터는 여러 항목을 한 번에 받아 **항목별로 구분된 판정 결과 목록**을 반환하는 배치 메서드를 지원해야 한다. 배치 내 일부 항목의 응답 파싱 실패가 다른 항목의 판정에 영향을 주지 않도록 항목 단위로 결과를 격리해야 한다(§4.2 배치 부분 실패 처리와 연동)

### 13.2 개인정보 마스킹 정책 적용
- PRD §6.2의 외부 API 전송 정책(패턴 기반 자동 마스킹: 주민등록번호·전화번호·이메일 자동 치환, 성명은 사용자 선택)을 적용한다
- **적용 범위**: 마스킹은 판정 대상 셀 값뿐 아니라, API로 전송되는 **모든 값**(§4.2의 인접 컬럼 컨텍스트 값, 컬럼 population 샘플 값 포함)에 동일하게 적용한다. 어떤 값이든 외부로 전송되기 전에는 예외 없이 마스킹 정책을 거친다

### 13.3 장애 폴백 및 PRD §6.4 예외 규정
- AI API **서비스 전체** 장애·타임아웃 발생 시 2단계는 자동 생략되며, 1단계 규칙기반 검증 결과만으로 정상 진행한다
- 이 경우 해당 항목은 "정상(AI 미검증)" 상태로 표시하여 담당자가 참고할 수 있도록 한다(본 문서 §6.2)
- 서비스는 정상이나 **개별 항목의 응답 파싱만 실패**하는 경우는 본 절이 아니라 §4.2.1을 따르며, "정상(AI 미검증)"이 아니라 **"이상(AI 재검증 실패)"**으로 처리한다(사유 상세 노출)
- **PRD §6.4 개정(제안) 문구**: "정성·정량 데이터 검토(F2-2)의 2차 AI 재검증 단계는 AI 장애 시 자동 생략되며, 규칙기반 1차 검증 결과만으로 정상 진행된다. 이 경우 해당 항목은 'AI 미검증' 상태로 표시된다. 이미지 검토(F2-2)는 원래부터 AI에 의존하지 않으므로 영향 없음."
