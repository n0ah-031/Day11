## 1. 개요

### 1.1 배경
기관 내에서 여러 부서·개인이 작성한 산출물(엑셀, 한글 문서)을 하나로 취합하는 업무는 반복적이고 오류가 잦다. 양식이 통일되지 않은 채 회신된 파일을 담당자가 수작업으로 검토·병합하면서 누락, 중복, 형식 불일치 문제가 상시 발생한다.

### 1.2 목적
취합앱은 **양식 생성 → 회신 파일 검토 → 취합·병합**의 전 과정을 하나의 웹 서비스로 제공하여, 취합 담당자의 수작업을 최소화하고 결과물의 정합성을 보장한다.

### 1.3 핵심 가치
1. 생성형 AI를 활용한 배포용 양식의 신속한 생성과 작성 기준 표준화
2. 검토 결과를 직관적인 화면으로 제시하여 취합 전 문제 파일·문제 값을 사전 식별
3. 엑셀·한글(hwpx) 등 국내 실무 환경에 맞춘 취합·병합 자동화, 부서 특성에 맞는 다양한 합성 방식 지원

### 3.1 사용자 유형
- **User (일반 사용자)**: 취합 실무 담당자. 양식 생성, 파일 업로드, 검토, 취합 실행, 결과 다운로드.

## 4. 기능 요구사항

### F1. 생성 — AI 기반 배포 양식 생성

생성형 AI(외부 상용 API 연동)를 활용해 배포용 취합 양식을 만드는 기능.

| ID | 요구사항 | 우선순위 |
|---|---|---|
| F1-1 | 사용자가 자연어 프롬프트로 만들고자 하는 양식을 최초 요청 | P0 |
| F1-2 | **문답형 정보 수집**: AI가 대상 부서, 포함 항목뿐 아니라 항목별 입력 형식(날짜 표기, 숫자 단위 등)·필수/선택 여부까지 PRD 작성 인터뷰처럼 세부적으로 순차 질문하여 확인. **(v1.5) 질문 내용·순서·사양 완료 판정은 고정 시나리오 없이 AI가 주도하되, 시스템이 필수 사양(양식명·항목 목록·항목별 입력 형식·필수 여부)의 완결성을 별도 검증하여 미비 시 생성을 차단한다.** 수집된 사양은 화면에 실시간으로 표시하고(항목의 추가·수정·삭제 반영), 사양이 충분히 모이기 전에는 생성 버튼을 비활성화. **(v1.5) 진행 상태 표시는 고정 단계 비율이 아닌, 확인된 사양 주제 기반의 근사치로 표시** | P0 |
| F1-3 | 문서 파일(hwpx, xlsx, png 등)을 컨텍스트로 첨부하여 양식 생성 요청. **(v1.5) 이미지(png) 첨부는 자동 마스킹이 적용되지 않음을 고지한 후 허용(§6.2 예외 규정 참조)** | P0 |
| F1-4 | 결과물 렌더링: 배포용 양식 파일로 출력. **(v1.5) xlsx 출력을 우선 구현하고, hwpx 출력은 P1로 조정하여 M3 후반에 배치(MVP 범위 내 유지)** | xlsx P0 / hwpx P1 |
| F1-5 | 양식 생성 시 **데이터별 작성 기준 수립**: 날짜 형식, 이름 형식 등 각 필드의 입력 규칙을 정의하고 양식과 함께 저장 | P0 |
| F1-6 | 생성된 양식·작성 기준을 프로젝트에 저장, 이후 취합 단계(F2-5 전처리)에서 자동 참조 | P1 |
| F1-7 | 대화형 수정: 생성 완료 후 추가 프롬프트로 양식을 반복 수정 (F1-2 문답형 수집과는 별개로, 생성 이후 단계에 적용) | P1 |

**작성 기준(스키마) 예시**
- 날짜: `YYYY-MM-DD`
- 성명: 성+이름 붙여쓰기, 공백 제거
- 금액: 숫자만, 천 단위 구분 없음
- 부서명: 사전 정의된 코드표 기준

**양식 생성 아키텍처 개요 (F1, v1.5 신규)**

- 양식의 내용·레이아웃·서식은 **AI(LLM)가 직접 저작**한다. 시스템은 저작 결과에 대한 안전 검증(구조 한도, 수식 제한 등)과 파일 변환·저장만 담당하며 자체 레이아웃 로직을 갖지 않는다 — 제목행·병합셀·안내문 등 다양한 레이아웃 요구에 대응하기 위한 유연성 확보 목적
- AI 저작 특성상 동일 사양에서도 실행마다 결과가 달라질 수 있으므로, 생성 직후 **미리보기를 필수 노출**하고 대화형 수정(F1-7)으로 교정하는 흐름을 기본 UX로 한다
- 사양 스키마, 검증 규칙, API 설계 등 구현 상세는 별도 문서(**생성기능_기술명세_v0.1.md**)에서 관리

### F2. 취합 — 엑셀

여러 개의 회신 엑셀 파일을 분석(구조 인식)하고 검토·취합하는 핵심 기능.

**처리 파이프라인:** 업로드 → ① 검토 → ② 이상판별(검토 결과 표시) → ③ 범위 확인 → ④ 전처리 → ⑤ 합성 모드 선택 → ⑥ 합성 → 다운로드

| ID | 요구사항 | 우선순위 |
|---|---|---|
| F2-1 | 다중 엑셀 파일 업로드 및 구조 인식(시트, 헤더, 데이터 영역 자동 감지) | P0 |
| F2-2 | **검토**: 파일별로 정합성·누락·중복·충돌 여부 자동 점검. 데이터 유형(정성적·정량적·이미지)별로 1차 규칙 기반 검증을 수행하고, 정성적·정량적 데이터는 1차 통과 시 2차 AI(LLM) 재검증까지 거쳐 최종 판정한다(이미지는 1차 규칙 검증만 수행). 상세 판정 기준·알고리즘은 별도 문서(F2_취합_엑셀_상세정의.md) 참조 | P0 |
| F2-3 | **이상판별(검토 결과 표시)**: F2-2 검토에서 발생한 이상값을 파일 리스트 + 상태 표시(정상/이상 이진화) UI로 사용자에게 출력. 상태값에 마우스오버·클릭 시 문제 위치(파일명·시트명·셀 위치)와 사유를 노출 (상세 명세는 아래 참조) | P0 |
| F2-4 | **취합 진행 여부 확인 UI**: F2-3의 파일 리스트에서 체크박스로 취합 범위를 선택. 체크박스 선택/해제에 따라 "선택된 파일 n/m건"과 하단 진행 버튼 "취합을 진행하시겠습니까 (n/m)" 문구가 실시간으로 갱신됨 | P0 |
| F2-5 | **데이터 전처리**: 생성 단계의 작성 기준이 있으면 준용, 없으면 사용자가 데이터별 기준을 설정하여 형식을 통일 | P0 |
| F2-6 | **합성 모드 A — 원본 보존형**: 부서별 시트를 원본 그대로 유지하며 하나의 파일로 합성 (결과 시트 개수 = n×m, 부서 수×시트 수) | P0 |
| F2-7 | **합성 모드 B — 시트명 통합형**: 시트명이 동일한 시트끼리 부서별 데이터를 세로로 누적, "부서" 구분 컬럼 추가 (결과 시트 개수 = m) | P0 |
| F2-8 | **합성 모드 C — 그룹(카테고리) 통합형**: 사전 정의된 그룹 매핑(시트명→그룹명)에 따라 여러 시트를 그룹 단위로 재편성해 통합, "구분"(원 시트명)·"부서" 컬럼 추가 (결과 시트 개수 = k, 그룹 수) | P1 |
| F2-9 | **합성 모드 D — 원본보존+요약형**: 모드 B의 결과 구조를 유지하며, 부서별 핵심 지표를 SUMIF/SUMIFS 수식으로 집계하는 "종합요약" 시트를 최상단에 추가 (결과 시트 개수 = m+1) | P1 |
| F2-10 | 확장 합성 모드 — v2 백로그: ① 키 기준 병합(동일 키 행 매칭 후 열 결합), ② 요약 집계(피벗형), ③ 세로 누적 → 가로 전개 변환. 출시 후 사용자 피드백으로 케이스 추가 수집 | P2 |
| F2-11 | 취합 결과 리포트: 처리 건수, 제외 파일, 수정된 항목 요약 제공 | P1 |
| F2-12 | **오류 리포트 다운로드(반려 회신용)**: 파일별 오류 내역(유형·위치·사유)을 문서로 내려받아 제출자에게 수정 요청 시 활용 | P0 |

**합성 모드 상세 정의 (F2-6~F2-9)**

| 모드 | 결과 구성 | 추가 컬럼 | 필요 입력 | 주요 예외 처리 |
|---|---|---|---|---|
| A. 원본 보존형 | {부서명}_{원본시트명} 시트 n×m개 | 없음 (원본 그대로) | 없음 | 시트명 31자 초과 시 자르기 규칙 적용 |
| B. 시트명 통합형 | 동일 시트명 기준 시트 m개 | 부서 | 없음 | 부서 간 헤더 불일치 시 오류 시트로 분리 |
| C. 그룹 통합형 | 그룹 매핑 기준 시트 k개 | 구분(원 시트명), 부서 | 그룹 매핑(시트명→그룹명) | 매핑에 없는 시트는 "미분류" 그룹 자동 편입 후 경고 |
| D. 원본보존+요약형 | 모드 B 구성(m개) + 종합요약 시트 1개 | 모드 B와 동일 + 요약 시트 지표 합계 | 요약 대상 지표(컬럼) | 지정 컬럼 부재 시 "N/A" 처리, 수치는 수식 기반(하드코딩 금지) |

**검토 아키텍처 개요 (F2-2, v1.4 신규)**

- 정성적(텍스트)·정량적(숫자) 데이터: **1차 규칙 기반 알고리즘 검증** → 통과 시에만 **2차 AI(LLM) 재검증**을 거쳐 최종 판정. 어느 단계에서든 이상이 발견되면 즉시 확정하고 다음 단계로 넘어가지 않는다
- 이미지 데이터: 1차 규칙 기반 검증(확장자·용량·해상도·중복 등 구조/메타데이터 체크)만 수행하고 종료. 내용 기반 AI 분석(Vision)은 범위 밖(v2 검토)
- 이 구조는 §6.4의 "AI 장애 무관 정상 동작" 원칙에 예외를 발생시키며, 상세는 §6.4 및 잔여 리스크 노트 참조
- 판정 알고리즘, 데이터 유형별 세부 기준, 임계값, AI 프롬프트 설계 등 구현 상세는 팀원별 개발 담당 분야에 맞춰 별도 문서(**F2_취합_엑셀_상세정의.md**)에서 관리하며, 필요 시 유사한 상세 문서를 F1·F3에도 확장할 예정

**이상판별 UI 명세 (F2-3) — 파일 리스트 + 상태 표시 방식**

기획 원안의 성공/실패 체크리스트 UI를 확장한 형태로, 검토 결과 확인과 취합 범위 선택(F2-4)이 한 화면에서 이루어진다.

1. **파일 리스트**: 업로드된 파일을 행 단위로 나열. 각 행 = 체크박스(취합 포함 여부) + 파일명 + 상태 표시
2. **상태 표시 — 정상/이상 이진화 (v1.4)**: 화면에는 `정상` 또는 `이상` 두 가지만 표시한다. 기존의 정상/경고/오류 3단계 구분은 화면에 노출하지 않고 **내부 판정 등급**으로 유지하여 F2-4의 기본 선택 로직에만 사용한다(아래 "배지 등급 분류 기준" 참조)
3. **상세 노출(마우스오버/클릭)**: `이상` 상태값에 마우스오버 또는 클릭 시 해당 파일의 문제 상세를 노출 — 오류 유형(정합성/누락/중복/충돌), 위치(시트·셀 좌표), 사유를 항목별로 표시
4. **기본 선택 상태**: 내부 등급 기준 `정상`·`경고` 파일은 체크, `오류` 파일은 체크 해제 상태로 초기 표시(사용자가 변경 가능)
5. **오류 리포트 다운로드(F2-12)**: 화면 상단에 다운로드 버튼 제공. 파일별 또는 전체 오류 내역을 xlsx 리포트로 생성 — 제출자에게 반려·수정 요청 회신 시 그대로 첨부 가능한 형식(파일명, 시트, 셀 위치, 오류 유형, 사유, 작성 기준 열 포함)
6. **하단 진행 버튼**: "취합을 진행하시겠습니까 (선택/전체)" 형태로 선택 건수 실시간 반영

## 5. 사용자 시나리오 (핵심 플로우)

1. **양식 배포 준비** — 담당자 A가 로그인 후 "부서별 예산 집행 현황 양식 만들어줘"라고 프롬프트 입력, 기존 hwpx 참고 파일 첨부 → AI가 대상 부서·항목·입력 형식을 문답으로 확인 → xlsx 양식과 작성 기준 생성 → 미리보기 확인 후 필요 시 대화형 수정 → 다운로드하여 각 부서에 배포(배포 자체는 앱 외부에서 수행).2. **회신 취합** — 부서별 회신 파일 10개를 수집하여 일괄 업로드 → 시스템이 1차 규칙 검증과 2차 AI 재검증을 거쳐 검토 실행 → 파일 리스트에 정상/이상 상태로 표시(정상 7 / 이상 3), 이상 파일 3개의 상태값에 마우스오버하여 날짜 형식 오류·중복 행 상세(파일명·시트·셀 위치) 확인 → 오류 리포트를 다운로드해 해당 부서에 수정 요청 회신 → 우선 문제없는 7개만 체크(선택 건수 실시간 반영) → "취합을 진행하시겠습니까 (7/10)" 확인 → 전처리(작성 기준 자동 준용) → 합성 모드 선택 화면에서 시트명 통합형(모드 B)을 선택하고 결과 미리보기로 산출 형태 확인 → 합성 → 결과 다운로드.
3. **한글 병합** — 부서별 보고서 hwpx 15개 업로드 → 드래그로 목차 순서 지정 → 서버 비동기 병합 → 완료 알림 후 다운로드.

## 6. 비기능 요구사항

### 6.1 성능·용량
- 동시 사용자: 최대 500명 등록 기준, 동시 접속 50명 수준 무리 없이 처리
- 업로드 제한(확정): 파일당 50MB, 1회 취합 작업당 최대 30개 파일 / 총 500MB. 초과 시 안내 메시지 표시
- 엑셀 취합(파일 10개, 총 10만 행 이내): 동기 처리 3분 이내 목표, 초과 시 비동기 전환
- hwpx 병합: 기준 초과 시 비동기 작업 큐 처리, 진행률 표시

### 6.2 보안
- 전 구간 HTTPS(TLS 1.2+), 저장 데이터 암호화(at-rest)
- 비밀번호 해시 저장(bcrypt/argon2), 로그인 시도 제한
- **외부 AI API 전송 정책(확정)**: OpenAI GPT API 사용. 문서 데이터가 외부 API로 전송됨을 사용자에게 고지하고 동의 절차 마련. API 데이터 학습 미사용(zero data retention) 조건 적용. F2-2의 2차 AI 재검증에 전송되는 셀 값에도 동일한 마스킹·동의 정책을 적용
- **개인정보 자동 마스킹(확정)**: 외부 API 전송 전 패턴 기반(정규식) 마스킹 적용 — 주민등록번호·전화번호·이메일은 자동 치환, 성명 열은 사용자가 마스킹 여부 선택. AI 기반 개인정보 탐지는 v2 검토. **(v1.5 예외 규정)** 양식 생성 첨부 중 이미지(png)는 패턴 기반 마스킹을 적용할 수 없으므로, AI 전송 동의 절차에 "이미지는 자동 마스킹이 적용되지 않음"을 명시적으로 고지하고 개인정보가 포함된 이미지 첨부를 지양하도록 안내한 후 허용한다(F1-3)
- **자유 가입 방침에 따른 통제**: 가입 자체는 제한하지 않되, Admin이 신규 가입 알림·계정 정지·삭제 권한으로 사후 통제하고 전체 사용 로그를 기록
- 사용자별 데이터 격리(타 사용자 파일 접근 불가), Admin 접근 로그 기록

### 6.3 데이터 보관
- 업로드 파일·결과물은 **90일 보관 후 자동 삭제(확정)**, Admin이 정책 변경 가능
- 사용자 수동 삭제 지원, 병합용 임시 파일은 작업 완료 즉시 삭제

### 6.4 가용성·운영
- 클라우드 인프라 기반, 목표 가용성 99.5%
- AI API 장애 시: 양식 생성(F1)은 기능 저하 모드로 안내. 엑셀 검토(F2-2)의 1차 규칙 기반 검증과 이상판별·취합(F2), 한글 병합(F3)은 AI 장애와 무관하게 정상 동작을 유지한다. **단, F2-2의 2차 AI 재검증(정성적·정량적 데이터 한정)은 장애 시 자동 생략되며, 해당 항목은 1차 검증 결과만으로 "정상(AI 미검증)"으로 표시되어 취합 진행이 차단되지 않는다(v1.4 예외 규정, 상세는 F2_취합_엑셀_상세정의.md)**

### 6.5 기술 결정 사항
- **AI 벤더**: OpenAI GPT API. 단, API 호출부는 어댑터 계층으로 분리 구현하여 추후 벤더 교체 가능성 확보. F2-2의 2차 AI 재검증도 동일 어댑터·벤더를 사용
- **hwpx 처리**: hwpx는 ZIP+XML(OWPML) 개방형 포맷이므로 직접 파싱·병합 방식으로 개발. M1에서 PoC를 수행하고, 복잡 서식(개체·스타일 충돌) 병합 품질이 기준 미달일 경우 상용 SDK(한컴 등) 전환 여부를 M1 종료 시점에 판단(전환 게이트)
- **양식 생성 저작 주체(v1.5 신규)**: 생성(F1)의 양식 내용·레이아웃·서식은 AI가 직접 저작하고, 시스템은 저작 결과의 안전 검증과 파일 변환·저장만 담당한다(자체 레이아웃 로직 없음). 상세는 생성기능_기술명세_v0.1.md(문서 버전 v0.2) 참조

### 잔여 리스크 노트
- **자유 가입 정책**: 내부용 서비스임에도 가입 제한이 없어 외부인 가입이 기술적으로 가능. Admin의 신규 가입 모니터링을 운영 절차로 반드시 수립할 것. 외부 가입 사례가 실제 발생하면 도메인 제한 도입을 재검토
- **hwpx 전환 게이트**: M1 말 PoC 판정이 '전환'으로 나올 경우 상용 SDK 라이선스 계약 기간이 M2 일정에 영향을 줄 수 있으므로, M1 중반에 후보 SDK 견적을 병행 확보해둘 것
- **모드 C 그룹 매핑 관리 방식**: 그룹 매핑(시트명→그룹명)을 화면 UI로 관리할지, 별도 설정 파일로 관리할지 design.md 단계에서 확정 필요
- **모드 D 요약 지표 정의**: 종합요약 시트에서 집계할 지표(컬럼)를 고정할지, 사용자가 매번 지정하게 할지 design.md 단계에서 확정 필요
- **F2 AI 재검증 신뢰도(v1.4 신규)**: 2차 AI(LLM) 재검증 판정은 확정적이지 않아 기본적으로 '경고' 등급으로만 처리하고 자동교정 대상에서 제외함. 실제 오탐률은 파일럿 테스트로 검증 필요
- **이미지 데이터 AI 분석(v1.4 신규)**: 이미지 셀 검토는 규칙 기반(확장자·용량·해상도·중복 등) 구조 체크로 한정하며, 내용 기반 AI Vision 분석은 v2에서 검토
- **F1 AI 직접 저작 품질 편차(v1.5 신규)**: 양식 내용을 AI가 직접 저작하므로 동일 사양에서도 실행마다 결과가 달라질 수 있음. 미리보기 필수 노출 + 대화형 수정(F1-7) 흐름으로 흡수하되, 대표 시나리오 기반 품질 테스트(골든 테스트)를 출시 전 수행할 것
- **F1 AI 주도 문답의 완료 판정(v1.5 신규)**: 사양 완료 판정을 AI가 주도하므로 조기 완료·과잉 질문 가능성 존재. 시스템의 완결성 검증과 문답 턴 수 상한으로 보강하며, 실제 판정 품질은 파일럿 테스트로 검증 필요
- **F1 이미지 첨부 개인정보(v1.5 신규)**: png 첨부는 마스킹이 적용되지 않는 유일한 외부 전송 경로. 고지 문구의 실효성을 모니터링하고, 문제 사례 발생 시 v2의 AI 기반 개인정보 탐지(Vision) 도입을 앞당겨 검토

## 11. UI 설계 참고 (화면 목업 반영)

핵심 화면 10개 전체에 대해 디자인 시안(목업)을 진행했다. 상세 비주얼과 인터랙션 명세는 design.md에서 별도 관리하고, 본 절에서는 PRD 요구사항과의 연결점만 요약한다.

| 화면 | 관련 ID | 핵심 설계 반영 사항 |
|---|---|---|
| 화면 1 · 로그인 및 회원가입 | F4-1 | 로그인 ID를 사번(직원번호)으로 변경. 회원가입 시에는 비밀번호 재설정용 이메일을 별도 필드로 추가 수집. 로그인/회원가입 탭 전환에 따라 확인 필드가 자동으로 나타나고 사라지도록 구성 |
| 화면 2 · 일반 User 홈 | F4-4, F1, F2-1 | 상단 탭바를 '생성'/'취합'으로 구성, 기본 선택 탭은 '생성'. '생성' 탭은 GPT/Gemini 홈 화면처럼 인사말 + 큰 프롬프트 입력창 + 추천 프롬프트 칩으로 구성하고, '취합' 탭 선택 시 화면 4(엑셀 업로드)로 전환. 로그인 후 모든 User가 진입하는 기본 화면 |
| 화면 3 · AI 양식 생성 | F1-1~F1-2, F1-7 | 단발성 프롬프트 입력 대신 챗봇형 문답 UI로 변경 — 대상 부서·포함 항목뿐 아니라 항목별 입력 형식(날짜 표기, 단위 등)까지 PRD 인터뷰처럼 심층 질문. 우측에 지금까지 확인된 사양을 카드로 실시간 표시하고, 사양이 모두 채워지기 전에는 "생성" 버튼이 비활성 상태로 유지됨. **(v1.5)** 문답이 AI 주도(고정 단계 없음)로 변경됨에 따라 진행률은 확인된 사양 주제 기반 근사치로 표시하고, 사양 카드는 누적 방식이 아닌 매 턴 전체 갱신(항목 수정·삭제 반영). 생성 완료 후 결과 미리보기와 대화형 수정 입력창을 같은 화면에 노출 |
| 화면 4 · 엑셀 업로드 및 구조 인식 | F2-1 | 업로드 즉시 파일별로 인식된 시트 목록과 상태 표시(인식 완료/확인 필요)를 노출 |
| 화면 5 · 검토 결과 및 취합 범위 선택 | F2-3, F2-4 | 체크박스 선택/해제 시 "선택된 파일 n/m건"과 하단 진행 버튼 문구가 정적 텍스트가 아니라 실시간으로 연동되도록 수정 반영. **(v1.4)** 파일 상태 표시는 정상/이상 2종으로 단순화되며, 상태값 마우스오버·클릭 시 상세 사유(파일명·시트명·셀 위치 포함)를 노출 |
| 화면 6 · 합성 모드 선택 | F2-6~F2-9 | 모드 A/B/C/D를 카드형 선택 UI로 제시. 각 모드 설명을 완전한 문장으로 다시 작성하고, 카드 선택 시 하단에 해당 모드로 취합했을 때 생성되는 시트 탭 목록과 예시 데이터를 보여주는 "결과 미리보기" 영역을 추가 |
| 화면 7 · 취합 결과 및 다운로드 | F2-11, F2-12 | 지표 카드(취합 파일/생성 시트/자동 수정/제외 파일)에 아이콘을 적용. 결과 파일·제외 파일 카드는 반응형 2열 그리드로, 자동 수정 항목은 아이콘 기반 카드 그리드로 배치. 지표 카드 그리드는 CSS 미디어 쿼리 기준폭으로 좁은 폭에서는 2×2, 넓은 폭에서는 4×1로만 전환되도록 고정(auto-fit 사용 시 발생하는 3+1 같은 중간 배치 방지) |
| 화면 8 · 한글(hwpx) 병합 | F3-1, F3-2 | 업로드된 파일 개수를 실시간 표시하고, 목록의 각 행을 실제로 드래그하면 병합 순서가 즉시 바뀌도록 구현(정적 그립 아이콘이 아닌 동작하는 재정렬) |
| 화면 9 · 작업 이력 | F4-2 | 검색창과 유형 필터(전체/엑셀/hwpx) 클릭 시 목록이 실제로 필터링되도록 구성. 일반 User의 로그인 후 랜딩 화면 후보로도 검토 중(잔여 리스크 노트 참조) |
| 화면 10 · Admin 콘솔 | F4-3 | 지표 카드·계정 목록·정책 입력 영역의 구조와 여백을 재검토. 보관 기간 입력칸이 잘려 보이던 문제를 너비·패딩 조정으로 수정하고, 계정 이메일이 길어질 경우 말줄임 처리되도록 보완. 화면 2가 일반 User 홈으로 재정의되면서, Admin의 시스템 사용 현황 대시보드는 본 화면 하나로 통합됨 |
